# Contributing

## Contribution Scope

Contributions are welcome for:
- chapter structure
- technical precision
- diagrams and tables
- glossary discipline
- references and bibliography
- editorial corrections
- source validation
- quality scripts

## Rules

1. Do not add hype language.
2. Define terms before using them extensively.
3. Preserve distinction between model, agent, tool, workflow, runtime, memory, orchestration, and protocol.
4. Add trade-offs and failure modes when proposing new material.
5. Record substantive sources in `refs/source-log.md`.
6. Update glossary or references when introducing new core terms.
7. Keep pull requests narrow and reviewable.

## Pull Request Expectations

A good pull request should include:
- scope statement
- files changed
- rationale
- open questions
- review guidance
- note on whether glossary, references, or appendices were updated
