# Citation Policy

## Purpose
Keep references proportionate, consistent, and auditable.

## Rules
- Record substantive technical claims that need support.
- Distinguish standards, academic work, vendor documentation, benchmarks, and field observations.
- Do not present opinion as established fact.
- Maintain one reference style consistently across the manuscript.
- Add unresolved source needs to `refs/source-log.md`.

## Minimum expectations
For chapters making strong technical claims, track:
- definitional sources
- architectural sources
- evaluation sources
- operational or governance sources where relevant
