# Real-World Best Practice

This repository is optimized for a practical writing reality:

- source material is uneven
- chapter quality varies
- terminology drifts over time
- prompts produce mixed output unless controlled tightly
- evidence gathering is often delayed unless formalized

## What works in practice

### 1. Establish a stable conceptual vocabulary early
For Agentic AI, the critical early distinctions usually include:
- model vs agent
- agent vs assistant
- tool use vs workflow automation
- memory vs state
- orchestration vs planning
- single-agent vs multi-agent

### 2. Draft one chapter at a time
Wide shallow drafting creates terminology debt.
Narrow deeper drafting improves overall quality faster.

### 3. Separate creation from criticism
First draft the chapter.
Then critique it with the review prompts.
Then rewrite.

### 4. Preserve open questions explicitly
Do not hide uncertainty. Track it.

### 5. Maintain the glossary continuously
The glossary is part of the writing system, not a final appendix task.

### 6. Keep references and claims linked
A technically serious book weakens quickly when source tracking is delayed.
