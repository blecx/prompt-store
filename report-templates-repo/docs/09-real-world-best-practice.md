# Real-World Best Practice

## What distinguishes a strong real-world report

A world-class report is not merely comprehensive. It is **useful under decision pressure**.

That means it should:

- define the decision or problem clearly
- prevent misreading through explicit scope boundaries
- present findings in a hierarchy of importance
- include trade-offs rather than pretending a single option is obviously best
- state what remains unknown
- make next steps concrete

## Recommended defaults

- write the executive summary last
- use tables for comparisons and risk logic
- use appendices for detail that would interrupt the main argument
- document assumptions even when they are uncomfortable
- prefer explicit limitations over false confidence

## Typical section sequence for high-value reports

1. executive summary
2. context, objective, and scope
3. methodology and evidence basis
4. key findings
5. analysis and implications
6. options and trade-offs
7. recommendations
8. roadmap or implementation path
9. risks and limitations
10. references and appendices
