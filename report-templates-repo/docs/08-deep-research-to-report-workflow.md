# Deep-Research-to-Report Workflow

## Why a dedicated workflow is needed

Deep-research output is often rich but not yet usable as a professional report. It may contain excellent material but still miss:

- decision framing
- report logic
- section discipline
- trade-off synthesis
- implementation implications

## The transformation sequence

1. create or refine the deep-research brief
2. obtain the deep-research output
3. normalize the output with the deep-research-to-report skill prompt
4. force explicit structure
5. review for gaps, overclaims, and contradictions
6. move final content into the report skeleton

## Best-practice instruction for transformation

Tell the model to:

- preserve substance
- remove repetition
- expose gaps instead of hiding them
- distinguish evidence from interpretation
- turn findings into decision-ready implications
- avoid hype and generic filler
