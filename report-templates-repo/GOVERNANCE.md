# Governance

This repository treats report generation as a controlled knowledge-production activity.

## Quality principles

- reports must be clear about evidence status
- reports must separate what is known from what is inferred
- reports must state scope boundaries
- reports must not hide uncertainty
- reports must be useful for decisions, not only for reading

## Review principles

Every major report version should pass at least these reviews:

- structural completeness review
- evidence-gap review
- contradiction/consistency review
- recommendation rigor review
- novice clarity review when the report targets mixed audiences
