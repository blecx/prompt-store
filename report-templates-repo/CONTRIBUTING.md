# Contributing

## Objective

Contributions should improve the report system in one of four ways:

- better structure
- better guidance for novices
- stronger evidence discipline
- stronger review and quality control

## Rules

- prefer explicit structure over stylistic flourish
- separate findings from interpretation
- avoid adding prompts that merely sound sophisticated without controlling quality
- any new prompt should state expected output structure and failure modes to avoid
- any new template should include placeholders for assumptions, limitations, and evidence

## Pull request expectations

A good change should explain:

- what problem it solves
- who benefits
- whether it changes workflow or only wording
- how a novice user should use it
