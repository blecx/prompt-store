# Report Workflow

## End-to-end flow

1. **Define the assignment**
   - audience
   - objective
   - decision to support
   - scope
   - constraints

2. **Prepare the intake**
   - use `templates/deep-research-request-template.md`
   - define questions, evidence needs, exclusions, and desired output

3. **Create the outline**
   - use `prompts/specialized/outlines/report-outline-from-brief.md`
   - verify that the outline matches the decision context

4. **Collect or transform content**
   - from fresh research, notes, interviews, or deep-research output
   - use the quick-use prompt when speed matters

5. **Draft report sections**
   - findings first
   - analysis second
   - recommendations third
   - roadmap and risks after that

6. **Run review passes**
   - evidence gaps
   - contradiction check
   - novice clarity
   - lector review

7. **Assemble and publish**
   - run scripts
   - finalize front matter and appendices
   - generate the final report package
